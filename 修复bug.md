二、 用户交互体验 (User Experience - UX)
目前的 UI 主要是“显示 + 拖拽”，在易用性上还有很大提升空间。

1. 拖拽时的实时冲突提示 (Real-time Feedback)
痛点：目前 handleDrop 后请求 API，后端计算完才弹窗报错。用户体验是割裂的。

改进：

当你拖拽某一节课（例如张老师的课）时，高亮显示所有张老师已经有课的格子（标红）。

在前端维护一份轻量级的 teacher_schedule 映射，拖拽悬停时，如果目标位置老师有冲突，直接将鼠标样式变为“禁止”，并显示 Tooltip：“张老师此处已有课 (3班)”。

2. 撤销/重做机制 (Undo/Redo)
痛点：排课和调课是高频试错过程。一旦手滑把一节排得很好的课拖走了，找不回来会很抓狂。

改进：

前端维护一个 historyStack。每次 API 调用成功后，将上一版 schedule 压栈。

提供 Ctrl+Z 快捷键或界面按钮回退。

3. Excel 导入配置 (不再依赖手动录入)
痛点：目前的“科目与师资设置”需要一条条添加。如果学校有 50 个老师、12 个班级，初始化工作量太大。

改进：

增加“上传 Excel 初始化”功能。

Excel 模板包含：[科目, 每周节数, 老师名单(逗号分隔), 教室限制]。

后端使用 pandas 读取并直接生成 config json。

4. 智能调课建议 (Smart Suggestions)
痛点：手动调课时，用户不知道把课挪到哪里合适。

改进：

点击某一节课，系统高亮显示所有 “可行的时间槽”（即：该班空闲 + 该老师空闲 + 教室空闲）。这会极大地降低调课难度。
